# 交付与新机器部署

## 1. 目标

这份文档用于说明一台新 Windows 机器如何从源码跑通 UE5.6 + LocalTTS + OmniVoice。

当前推荐路径：

```text
安装 UE5.6
-> 安装 Python
-> 运行 Setup_TTS_Service.bat
-> 运行 Build_UE56_TTSHost.bat
-> 打开 UE56_TTSHost.uproject
-> Play 后点击 Start / Health / Speak / Gen
```

## 2. 必要环境

- Windows。
- Unreal Engine 5.6。
- Visual Studio 2022，包含 C++ 游戏开发组件。
- Python 3.11 或 3.12，且命令行可执行 `python`。
- NVIDIA 显卡和可用 CUDA 环境，当前默认 `LOCAL_TTS_DEVICE=cuda`。
- 首次下载模型需要网络，模型较大，需要等待。

## 3. 初始化 Python 服务环境

在项目根目录运行：

```bat
D:\project\UETTsProject\Setup_TTS_Service.bat
```

脚本会创建：

```text
D:\project\UETTsProject\Services\tts_service\.venv
```

插件默认会使用这个 Python：

```text
Services\tts_service\.venv\Scripts\python.exe
```

如果 Python 没装好，脚本会提示无法创建虚拟环境。先在命令行确认：

```bat
python --version
```

## 4. 编译 UE 工程

推荐只运行项目根目录的脚本：

```bat
D:\project\UETTsProject\Build_UE56_TTSHost.bat
```

不推荐在 Visual Studio 里生成整个解决方案，因为 UE5.6 自带的一些 C# 工具项目可能触发 `Magick.NET` NuGet 漏洞审计警告，并被当成错误。

当前脚本已经设置：

```bat
NuGetAudit=false
WarningsNotAsErrors=NU1901;NU1902;NU1903;NU1904
```

如果编译失败，优先找输出里的第一条真实 C++ 或 UBT 错误。

## 5. 打开和测试工程

打开：

```text
D:\project\UETTsProject\UE56_TTSHost\UE56_TTSHost.uproject
```

进入 PIE 运行后，测试 Actor 按钮含义：

- `Start`：启动本地服务。
- `Health`：检查服务状态。
- `Speak`：生成 wav 并播放。
- `Gen`：只生成 wav，不播放，给数字人链路测试。
- `Stop`：停止当前播放。

首次启动服务时，OmniVoice 会加载模型。看到服务窗口出现 `service startup complete` 后再测试 `Speak` 或 `Gen`。

## 6. 模型和缓存目录

服务运行产物：

```text
Services\tts_service\cache
Services\tts_service\logs
Services\tts_service\models
Services\tts_service\.venv
Services\tts_service\vendor
```

这些目录不提交 git。

如果想指定 Hugging Face 模型缓存目录，可以设置：

```bat
set LOCAL_TTS_HF_CACHE_DIR=D:\models\huggingface
```

常用环境变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `LOCAL_TTS_HOST` | `127.0.0.1` | 服务监听地址。 |
| `LOCAL_TTS_PORT` | `50021` | 服务端口。 |
| `LOCAL_TTS_MODEL_NAME` | `k2-fsa/OmniVoice` | OmniVoice 模型名。 |
| `LOCAL_TTS_DEVICE` | `cuda` | 推理设备。没有 CUDA 时可尝试 `cpu`，但速度可能很慢。 |
| `LOCAL_TTS_DTYPE` | `float16` | 推理精度。 |
| `LOCAL_TTS_CACHE_DIR` | `Services\tts_service\cache` | 生成 wav 的缓存目录。 |
| `LOCAL_TTS_LOG_DIR` | `Services\tts_service\logs` | 服务日志目录。 |
| `LOCAL_TTS_LOAD_ASR` | `true` | 是否尝试加载 ASR。失败后服务会降级为无 ASR。 |
| `LOCAL_TTS_ASR_MODEL_NAME` | `openai/whisper-large-v3-turbo` | ASR 模型名，用于 clone 模式自动识别参考音频文本。 |
| `LOCAL_TTS_HF_CACHE_DIR` | 空 | Hugging Face 缓存目录。 |
| `LOCAL_TTS_REQUEST_TIMEOUT_SEC` | `300` | 单次 TTS 请求超时时间，单位秒。 |
| `LOCAL_TTS_EAGER_LOAD` | `true` | 服务启动时是否立即加载 OmniVoice 模型。 |

## 7. git 提交规则

应该提交：

- `Doc`
- `Services\tts_service\app`
- `Services\tts_service\requirements.txt`
- `Services\tts_service\run_server.py`
- `UE56_TTSHost\Config`
- `UE56_TTSHost\Plugins\LocalTTS\Source`
- `UE56_TTSHost\Source`
- `Build_UE56_TTSHost.bat`
- `Setup_TTS_Service.bat`

不应该提交：

- `.venv`
- `cache`
- `logs`
- `models`
- `vendor`
- UE `Binaries`
- UE `Intermediate`
- UE `Saved`
- UE `DerivedDataCache`

## 8. 常见问题

### Python executable not found

说明 `Services\tts_service\.venv\Scripts\python.exe` 不存在。

处理方式：运行 `Setup_TTS_Service.bat`。

### HTTP 502

通常说明服务端推理失败。先看服务窗口或：

```text
Services\tts_service\logs\tts_service.log
```

如果是 clone 模式，确认 `ReferenceAudioPath` 存在，并填写 `ReferenceText`。

### Timed out while waiting for LocalTTS service to become ready

通常是首次加载模型时间超过了 UE 等待时间，或模型下载失败。

处理方式：

- 等服务窗口出现 `service startup complete`。
- 再点击 `Speak` 或 `Gen`。
- 如果一直失败，检查网络和 Hugging Face 缓存。

### Magick.NET NuGet 漏洞警告

这是 UE5.6 引擎侧 C# 工具项目触发的 NuGet 审计问题，不是 LocalTTS 插件代码漏洞。

处理方式：使用 `Build_UE56_TTSHost.bat` 编译。

### Clone 模式提示 ref_text required

说明服务端 ASR 没有成功加载，不能自动识别参考音频内容。

处理方式：手动填写 `ReferenceText`。

### Design 模式提示 Unsupported instruct items

说明 `Instruct` 填成了自然语言描述，OmniVoice 只接受固定标签。

错误示例：

```text
A warm, natural Chinese female voice.
```

正确示例：

```text
female, chinese accent
```

或者：

```text
女，青年，中音调
```

英文标签用半角逗号加空格，中文标签用全角逗号，不要混用中英文。
