# 明日测试步骤

## 1. 测试前准备

推荐先运行一次编译：

```bat
D:\project\UETTsProject\Build_UE56_TTSHost.bat
```

如果显示 `Build succeeded.`，再打开 UE5.6 工程：

```text
D:\project\UETTsProject\UE56_TTSHost\UE56_TTSHost.uproject
```

## 2. 测试 Actor 按钮

测试 Actor 是 C++ 类 `ALocalTTSTestActor`，当前在编辑器中的显示名是 `LocalTTS Example Actor`。它是示例入口，不是正式业务蓝图入口，也不是必须存在一个同名蓝图。

如果场景里已经有继承自 `LocalTTSTestActor` 的蓝图实例，也可以继续用。蓝图继承只是方便你在场景里拖拽和保存默认参数，本质上仍然调用同一套 C++ 逻辑。

当前按钮含义：

- `Check Health`：检查本地 TTS 服务健康状态。
- `Start Service`：启动本地 TTS 服务。
- `Speak Example`：生成 wav 并在 UE 内播放。
- `Generate Example`：只生成 wav，不播放，给后续数字人口型和表情链路测试。
- `Stop Playback`：停止当前播放。

## 3. 普通播放测试

步骤：

1. 进入 PIE 运行模式。
2. 选中运行时的测试 Actor，而不是编辑器原始 Actor。
3. 点击 `Start Service`。
4. 等待服务窗口出现 `service startup complete`。
5. 点击 `Speak Example`。
6. 查看 `LastGeneratedWavPath` 是否出现新的 wav 路径。
7. 查看 `LastErrorMessage` 是否为空。

预期结果：

- `LastGeneratedWavPath` 指向 `Services\tts_service\cache\req_xxxxxx.wav`。
- UE 日志出现 `LocalTTS speech request finished`。
- UE 日志出现 `LocalTTS playback started`。
- 如果有音频设备，可以听到播放声音。

## 4. 只生成不播放测试

步骤：

1. 进入 PIE 运行模式。
2. 点击 `Start Service`。
3. 等待服务 ready。
4. 点击 `Generate Example`。
5. 查看 `LastGeneratedWavPath`。
6. 查看 `LastSpeechEventSummary`。

预期结果：

- 生成新的 wav 文件。
- 不触发 UE 内播放。
- `LastSpeechEventSummary` 包含 `request_id`、`text`、`wav`、`duration_sec`。
- `visemes=0`、`emotions=0` 是正常的，因为当前只是预留口型和表情轨道，还没有接分析器。

## 5. Stop 测试

步骤：

1. 输入一段较长文本。
2. 点击 `Speak Example`。
3. 在播放过程中点击 `Stop Playback`。

预期结果：

- 当前声音停止。
- UE 日志出现 `LocalTTS audio playback stopped and component released.`。
- 再次点击 `Speak` 仍能正常生成和播放。

## 6. 连续播放稳定性测试

步骤：

1. 进入 PIE 运行模式。
2. 点击 `Start Service`。
3. 连续点击 `Speak Example` 3 次，每次间隔等待播放开始。
4. 每次都观察 `LastGeneratedWavPath`、`LastTTSSummary` 和 UE 日志。

预期结果：

- 每次都生成新的 `req_xxxxxx.wav`。
- UE 不崩溃，不残留异常播放组件。
- UE 日志每次都能看到 `LocalTTS speech request finished` 和 `LocalTTS playback started`。

## 7. 3D 播放测试

正式项目推荐用蓝图节点 `生成并在 Actor 位置播放 Local TTS` 做 3D 播放。

建议步骤：

1. 在任意 Actor 蓝图中调用 `创建 Local TTS 自动模式请求`。
2. 调用 `生成并在 Actor 位置播放 Local TTS`，把当前 Actor 作为播放目标。
3. 观察 `OnAudioReady`、`OnFinished`、`OnError` 回调。

预期结果：

- 可以正常生成并在 Actor 位置播放。
- UE 日志出现 `LocalTTS playback started mode=actor`。

## 7.5 正式 UI 状态流接线模板

正式 UMG / Widget Blueprint 推荐不要直接用示例 Actor 按钮，而是按下面方式接正式节点。

建议 Widget 变量：

- `InputText`
  - 用户输入的朗读文本
- `StatusText`
  - 当前状态文本
- `ErrorText`
  - 错误提示文本
- `bSubmitEnabled`
  - 是否允许再次点击“生成并播放”
- `CurrentSpeakState`
  - 类型使用 `ELocalTTSSpeakAsyncState`

推荐按钮点击流程：

1. 点击“生成并播放”
2. 先调用 `Local TTS 是否忙碌`
3. 如果为 `true`
   - 直接显示“当前仍在处理上一条语音，请稍候”
4. 如果为 `false`
   - 调用 `创建 Local TTS 自动模式请求`
   - 把 `InputText` 写入请求
   - 调用 `生成并播放 Local TTS`

推荐异步回调接法：

1. `OnStateChanged`
   - 保存到 `CurrentSpeakState`
   - 调用 `获取语音异步状态文本`
   - 把结果写到 `StatusText`
   - 调用 `语音异步状态是否应禁用按钮`
   - 把结果取反后写到 `bSubmitEnabled`
2. `OnAudioReady`
   - 如果需要单独显示“准备完成”，可在这里把 `StatusText` 覆盖成“音频已就绪，准备播放”
3. `OnFinished`
   - 把 `StatusText` 设为“播放完成”
   - 把 `ErrorText` 清空
   - 恢复按钮可点击
4. `OnError`
   - 把错误信息写入 `ErrorText`
   - 把 `StatusText` 设为“语音流程失败”
   - 恢复按钮可点击

如果是“仅生成 Local TTS”：

- 同样监听 `OnStateChanged`
- 在 `OnAudioReady` 中读取返回的 `WAV Path`
- 把路径交给数字人、自定义播放器或后处理系统
- 这时不要等 `OnFinished` 才更新 UI，因为仅生成模式没有自动播放阶段

推荐 UI 文案对应关系：

- `Started` -> `思考中`
- `WaitingForService` -> `正在启动服务`
- `Generating` -> `正在生成语音`
- `AudioReady` -> `音频已就绪`
- `Playing` -> `正在播放`
- `Finished` -> `已完成`
- `Error` -> `发生错误`

## 8. 错误场景测试

### 8.1 缺失 wav 测试

建议方式：

1. 先执行一次 `Generate Example`，拿到 `LastGeneratedWavPath`。
2. 手动删除该 wav 文件。
3. 在蓝图或后续播放链路中尝试播放这个路径。

预期结果：

- 错误信息明确指出 wav 文件不存在。

### 8.2 坏 wav / 不支持格式测试

建议方式：

1. 准备一个非 `16-bit PCM` 的 wav 文件，或手动破坏一个测试 wav。
2. 通过后续蓝图或播放链路尝试播放。

预期结果：

- 错误信息明确指出格式、位深或声道数不受支持。

### 8.3 无音频设备测试

如果机器环境允许，可在禁用音频输出设备后测试。

预期结果：

- 错误信息明确指出当前世界没有可用音频设备。

## 9. 数字人链路当前怎么理解

目前数字人功能还不是“已经能驱动口型和表情”，而是已经预留了正确入口：

```text
Generate Local TTS Async
-> OnSpeechEventReady
-> FLocalTTSSpeechEvent
-> 未来接 LipSync / Emotion / MetaHuman Driver
```

也就是说，现在先保证语音结果不被播放器绑死。后续做数字人口型时，优先从 `FLocalTTSSpeechEvent.WavPath` 和 `Text` 开始接分析器。

## 10. 如果遇到问题

优先看这几个字段：

- `LastErrorMessage`
- `LastHealthSummary`
- `LastTTSSummary`
- `LastSpeechEventSummary`
- `LastGeneratedWavPath`

优先看这几类日志：

- `LocalTTS speech request finished`
- `LocalTTS speech request failed`
- `LocalTTS playback started`
- `LocalTTS audio playback stopped and component released`
- `LocalTTS generated speech event`
