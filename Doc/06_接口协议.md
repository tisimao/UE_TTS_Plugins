# 接口协议

## 1. 协议范围

本协议定义 UE 插件与本地 TTS 服务之间的通信格式。

第一版固定：

- 协议：HTTP
- 地址：`127.0.0.1`
- 端口：`50021`
- 编码：UTF-8
- 数据格式：JSON

## 2. 健康检查接口

### 请求

- 方法：`GET`
- 路径：`/health`

### 成功响应

```json
{
  "ok": true,
  "service": "tts_service",
  "status": "ready",
  "model": "CosyVoice"
}
```

### 失败语义

- 服务未启动
- 模型未加载完成
- 端口不可达

## 3. 语音生成接口

### 请求

- 方法：`POST`
- 路径：`/tts`

### 请求体

```json
{
  "text": "你好，这是测试语音。",
  "voice_id": "default_cn_female",
  "speed": 1.0
}
```

### 字段说明

- `text`
  - 类型：`string`
  - 必填
  - 第一版只接受中文文本

- `voice_id`
  - 类型：`string`
  - 可选
  - 第一版可先只支持一个默认值

- `speed`
  - 类型：`number`
  - 可选
  - 默认值：`1.0`

## 4. 成功响应

```json
{
  "ok": true,
  "request_id": "req_000001",
  "wav_path": "D:/project/UETTsProject/Services/tts_service/cache/req_000001.wav",
  "duration_ms": 2100
}
```

## 5. 失败响应

```json
{
  "ok": false,
  "request_id": "req_000001",
  "error_code": "MODEL_NOT_READY",
  "error_message": "TTS model is not ready."
}
```

## 6. 错误码约定

- `SERVICE_NOT_READY`
- `MODEL_NOT_READY`
- `INVALID_TEXT`
- `INVALID_VOICE`
- `INVALID_SPEED`
- `INFERENCE_FAILED`
- `WAV_EXPORT_FAILED`
- `INTERNAL_ERROR`

## 7. 插件侧处理规则

插件收到结果后按以下规则处理：

- `ok = true`
  - 开始加载并播放 wav

- `ok = false`
  - 直接回调蓝图错误事件

- HTTP 超时
  - 映射为 `SERVICE_NOT_READY`

- 返回文件不存在
  - 映射为 `WAV_EXPORT_FAILED`

## 8. 第一版约束

- 不做流式 chunk 协议
- 不返回 PCM 原始帧
- 不返回 viseme 数据
- 不返回情绪控制数据
