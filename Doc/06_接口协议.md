# 接口协议

## 1. 协议范围

本协议定义 UE 插件与本地 TTS 服务之间的通信格式。

第一版固定：

- 协议：HTTP
- 地址：`127.0.0.1`
- 端口：`50021`
- 编码：UTF-8
- 数据格式：JSON
- 模型：`OmniVoice`

## 2. 健康检查接口

### 请求

- 方法：`GET`
- 路径：`/health`

### 成功响应

```json
{
  "ok": true,
  "service": "tts_service",
  "status": "ready",
  "model": "OmniVoice",
  "supported_modes": ["auto", "clone", "design"]
}
```

### 失败语义

- 服务未启动
- 模型未加载完成
- 端口不可达

## 3. 语音生成接口

### 请求

- 方法：`POST`
- 路径：`/tts`

### 请求体

```json
{
  "text": "你好，这是测试语音。",
  "mode": "design",
  "language_id": "zh",
  "ref_audio": null,
  "ref_text": null,
  "instruct": "成年女性，普通话，自然播报",
  "duration": null,
  "speed": 1.0
}
```

## 4. 字段说明

- `text`
  - 类型：`string`
  - 必填
  - 待合成文本

- `mode`
  - 类型：`string`
  - 必填
  - 可选值：`auto`、`clone`、`design`

- `language_id`
  - 类型：`string`
  - 可选
  - 对应 `OmniVoice` 语言标识，例如 `zh`、`en`

- `ref_audio`
  - 类型：`string | null`
  - `clone` 模式必填
  - 参考音频绝对路径

- `ref_text`
  - 类型：`string | null`
  - `clone` 模式可选
  - 参考音频转写文本
  - 为空时允许服务侧触发自动转写
  - 若服务未启用 ASR，或离线环境下 ASR 模型不可用，则该字段变为必填

- `instruct`
  - 类型：`string | null`
  - `design` 模式必填
  - 用于描述说话人属性，例如性别、年龄、口音、声线

- `duration`
  - 类型：`number | null`
  - 可选
  - 输出目标时长，单位秒
  - `null` 或 UE 侧 `Duration = 0` 表示不发送该字段，由 OmniVoice 自行决定语音长度
  - 大于 `0` 时作为目标时长传给 OmniVoice，模型会尝试生成接近该秒数的音频，但不保证精确等于该值
  - 当前 UE 侧允许范围为 `0` 到 `60`，其中 `0` 代表不启用目标时长控制
  - 该字段不是请求超时时间，也不是响应里的 `duration_ms`
  - 响应里的 `duration_ms` 当前表示服务端生成耗时毫秒，不代表 wav 真实播放时长

- `speed`
  - 类型：`number | null`
  - 可选
  - 默认值：`1.0`
  - 若同时提供 `duration`，模型可能优先尝试满足目标时长，`speed` 的实际体感效果可能变弱

## 5. 模式规则

### `auto`

- 只要求 `text`
- `ref_audio`、`ref_text`、`instruct` 为空

### `clone`

- 要求 `text`
- 要求 `ref_audio`
- `ref_text` 可选
- 当服务侧 ASR 不可用时，`ref_text` 必填

### `design`

- 要求 `text`
- 要求 `instruct`

## 6. 成功响应

```json
{
  "ok": true,
  "request_id": "req_000001",
  "mode": "design",
  "wav_path": "D:/project/UETTsProject/Services/tts_service/cache/req_000001.wav",
  "sample_rate": 24000,
  "duration_ms": 2100
}
```

## 7. 失败响应

```json
{
  "ok": false,
  "request_id": "req_000001",
  "error_code": "INVALID_MODE",
  "error_message": "Mode design requires a non-empty instruct field."
}
```

## 8. 错误码约定

- `SERVICE_NOT_READY`
- `MODEL_NOT_READY`
- `INVALID_TEXT`
- `INVALID_MODE`
- `INVALID_REF_AUDIO`
- `INVALID_REF_TEXT`
- `INVALID_INSTRUCT`
- `INVALID_LANGUAGE`
- `INVALID_SPEED`
- `INVALID_DURATION`
- `INFERENCE_FAILED`
- `WAV_EXPORT_FAILED`
- `INTERNAL_ERROR`

## 9. 插件侧处理规则

插件收到结果后按以下规则处理：

- `ok = true`
  - 开始加载并播放 wav

- `ok = false`
  - 直接回调蓝图错误事件

- HTTP 超时
  - 映射为 `SERVICE_NOT_READY`

- 返回文件不存在
  - 映射为 `WAV_EXPORT_FAILED`

## 10. 第一版约束

- 不做流式 chunk 协议
- 不返回 PCM 原始帧
- 不返回 viseme 数据
- 不返回 MetaHuman 驱动数据
