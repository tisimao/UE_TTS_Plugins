# 插件设计

## 1. 插件目标

`LocalTTS` 是一个 UE5.6 Runtime 插件。

第一版只负责以下能力：

- 提供蓝图接口
- 自动启动本地 TTS 服务
- 向本地 TTS 服务发送结构化 `OmniVoice` 请求
- 接收返回的 wav 音频结果
- 在 UE 内部播放语音

第一版不负责：

- 口型驱动
- MetaHuman 联动
- 流式播放
- 完整的音色资产管理系统

## 2. 插件名称与模块

- 插件名：`LocalTTS`
- Runtime 模块：`LocalTTS`
- 后续 Editor 模块：`LocalTTSEditor`

当前阶段只要求 `LocalTTS` Runtime 模块可用。

## 4. 插件内部职责划分

### 4.1 服务管理

负责：

- 检查本地 TTS 服务是否运行
- 自动拉起本地 TTS 服务进程
- 停止本地 TTS 服务
- 提供服务状态查询

### 4.2 请求管理

负责：

- 发送 `POST /tts`
- 校验请求模式和必填字段
- 处理成功和失败响应
- 管理超时
- 对蓝图回调错误信息

### 4.3 音频播放

负责：

- 根据服务返回结果找到 wav 文件
- 在 UE 中创建可播放音频对象
- 播放音频
- 广播播放完成事件

### 4.4 蓝图接口

负责：

- 对外暴露稳定蓝图节点
- 用结构体承载复杂参数
- 屏蔽内部服务细节

## 5. 第一版蓝图节点

### `StartLocalTTS`

用途：

- 手动预热本地服务

输入：

- 无

输出：

- `Success`
- `ErrorMessage`

### `IsLocalTTSReady`

用途：

- 查询服务是否可用

输出：

- `bReady`

### `SpeakTextAsync`

用途：

- 发送结构化请求并播放语音

输入：

- `Request`

其中 `Request` 至少包含：

- `Text`
- `Mode`
- `LanguageId`
- `ReferenceAudioPath`
- `ReferenceText`
- `Instruct`
- `Duration`
- `Speed`

输出回调：

- `OnStarted`
- `OnAudioReady`
- `OnFinished`
- `OnError`

### `StopSpeaking`

用途：

- 停止当前播放

## 6. 建议类划分

第一版建议至少有以下类：

- `FLocalTTSModule`
  - 插件主模块入口

- `FLocalTTSHealthResponse`
  - 健康检查响应结构

- `FLocalTTSSpeakRequest`
  - 语音生成请求结构

- `FLocalTTSTTSResponse`
  - 语音生成响应结构

- `ULocalTTSBlueprintLibrary`
  - 简单蓝图函数入口

- `ULocalTTSCheckHealthAsyncAction`
  - 健康检查异步蓝图节点

- `ULocalTTSSpeakAsyncAction`
  - 异步蓝图节点

- `ULocalTTSSubsystem`
  - 统一服务和播放调度

- `FLocalTTSServiceProcess`
  - 本地服务进程管理

- `FLocalTTSHttpClient`
  - HTTP 请求封装

- `FLocalTTSRequestValidator`
  - 请求模式和字段校验

- `FLocalTTSWavLoader`
  - WAV 加载

- `ULocalTTSAudioPlayer`
  - 播放控制

当前已落地的第一批插件骨架：

- `FLocalTTSModule`
- `FLocalTTSHealthResponse`
- `FLocalTTSSpeakRequest`
- `FLocalTTSTTSResponse`
- `FLocalTTSHttpClient`
- `FLocalTTSRequestValidator`
- `FLocalTTSServiceProcess`
- `FLocalTTSWavLoader`
- `ULocalTTSSubsystem`
- `ULocalTTSBlueprintLibrary`
- `ULocalTTSCheckHealthAsyncAction`
- `ULocalTTSSpeakAsyncAction`
- `ULocalTTSAudioPlayer`

命名约束：

- 插件侧同样执行“类名就是文件名”
- 例如 `FLocalTTSServiceProcess -> FLocalTTSServiceProcess.h/.cpp`
- 例如 `ULocalTTSSpeakAsyncAction -> ULocalTTSSpeakAsyncAction.h/.cpp`

## 8. UE5.6 最小联调入口

当前宿主工程已经补了一个最小测试 Actor：

- `ALocalTTSTestActor`

对应文件：

- `UE56_TTSHost/Source/UE56_TTSHost/ALocalTTSTestActor.h`
- `UE56_TTSHost/Source/UE56_TTSHost/ALocalTTSTestActor.cpp`

用途：

- 不先手搓蓝图，也能直接验证本地服务链路
- 放进关卡后，可在 Details 面板直接点击按钮

可直接调用的测试按钮：

- `Health` -> `CheckLocalTTSHealth`
- `Start` -> `StartLocalTTSService`
- `Speak` -> `SpeakLocalTTSTest`
- `Stop` -> `StopLocalTTSSpeaking`

关键属性：

- `SpeakRequest`
  - 可直接在 Details 面板填写文本、模式、语言等参数
- `LastHealthResponse`
  - 查看最近一次健康检查结果
- `LastTTSResponse`
  - 查看最近一次语音生成结果
- `LastErrorMessage`
  - 查看最近一次错误信息

## 8. 第一版完成标准

- 宿主工程启用插件后可正常加载
- 蓝图中能看到 `SpeakTextAsync`
- `auto` 或 `design` 模式下输入中文后能成功发声
- 服务异常时能返回明确错误
