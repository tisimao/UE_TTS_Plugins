# 插件设计

## 1. 插件目标

`LocalTTS` 是一个 UE5.5 Runtime 插件。

第一版只负责以下能力：

- 提供蓝图接口
- 自动启动本地 TTS 服务
- 向本地 TTS 服务发送中文文本
- 接收返回的 wav 音频结果
- 在 UE 内部播放语音

第一版不负责：

- 口型驱动
- MetaHuman 联动
- 流式播放
- 多音色系统

## 2. 插件名称与模块

- 插件名：`LocalTTS`
- Runtime 模块：`LocalTTS`
- 后续 Editor 模块：`LocalTTSEditor`

当前阶段只要求 `LocalTTS` Runtime 模块可用。

## 3. 目录约定

```text
Plugins/LocalTTS
├─ LocalTTS.uplugin
├─ Content
├─ Resources
└─ Source
   └─ LocalTTS
      ├─ Public
      └─ Private
```

后续扩展为：

```text
Plugins/LocalTTS
└─ Source
   ├─ LocalTTS
   └─ LocalTTSEditor
```

## 4. 插件内部职责划分

### 4.1 服务管理

负责：

- 检查本地 TTS 服务是否运行
- 自动拉起本地 TTS 服务进程
- 停止本地 TTS 服务
- 提供服务状态查询

### 4.2 请求管理

负责：

- 发送 `POST /tts`
- 处理成功和失败响应
- 管理超时
- 对蓝图回调错误信息

### 4.3 音频播放

负责：

- 根据服务返回结果找到 wav 文件
- 在 UE 中创建可播放音频对象
- 播放音频
- 广播播放完成事件

### 4.4 蓝图接口

负责：

- 对外暴露稳定蓝图节点
- 保持参数简单
- 屏蔽内部服务细节

## 5. 第一版蓝图节点

### `StartLocalTTS`

用途：

- 手动预热本地服务

输入：

- 无

输出：

- `Success`
- `ErrorMessage`

### `IsLocalTTSReady`

用途：

- 查询服务是否可用

输出：

- `bReady`

### `SpeakTextAsync`

用途：

- 发送文本并播放语音

输入：

- `Text`
- `VoiceId`
- `Speed`

输出回调：

- `OnStarted`
- `OnAudioReady`
- `OnFinished`
- `OnError`

### `StopSpeaking`

用途：

- 停止当前播放

## 6. 建议类划分

第一版建议至少有以下类：

- `FLocalTTSModule`
  - 插件主模块入口

- `ULocalTTSBlueprintLibrary`
  - 简单蓝图函数入口

- `ULocalTTSSpeakAsyncAction`
  - 异步蓝图节点

- `ULocalTTSSubsystem`
  - 统一服务和播放调度

- `FLocalTTSServiceProcess`
  - 本地服务进程管理

- `FLocalTTSHttpClient`
  - HTTP 请求封装

## 7. 插件设置项

后续需要支持这些配置：

- TTS 服务路径
- Python 路径或运行时路径
- 服务端口
- 默认音色
- 默认语速
- 请求超时时间

第一版可以先硬编码，第二版再接入设置页。

## 8. 第一版完成标准

- 宿主工程启用插件后可正常加载
- 蓝图中能看到 `SpeakTextAsync`
- 输入中文后能成功发声
- 服务异常时能返回明确错误
