# UE TTS 插件需求冻结

## 1. 项目目标

本项目的目标是开发一个可复用的 Unreal Engine 5.5 插件，用于在 UE 场景中输入一段中文文本，并由本地 TTS 模型生成语音后直接播放。

本项目当前阶段的唯一核心目标是：

- 在 UE5.5 场景中，通过蓝图调用插件接口，输入一段中文文本，成功生成并播放中文语音。

## 2. 本阶段必须完成的能力

- 插件运行于 UE5.5。
- 插件以项目插件形式开发，后续可迁移到其他 UE 项目。
- 插件提供蓝图接口。
- 蓝图中可以输入一段中文字符串。
- 插件调用本地 TTS 模型生成音频。
- 插件在 UE 内部播放生成的音频。
- 整个流程不依赖云端服务。
- 模型推理默认运行在本地 NVIDIA GPU 上。

## 3. 本阶段明确不做的内容

- 不做数字人口型驱动。
- 不做 MetaHuman 表情驱动。
- 不做语音打断和插话控制。
- 不做流式边生成边播放。
- 不做多角色音色管理系统。
- 不做情绪风格控制系统。
- 不做商业发布级打包方案。
- 不做跨平台支持，本阶段仅面向 Windows。

## 4. 技术边界

- UE 版本固定为 UE5.5。
- 插件必须基于 C++ 开发，不能只做纯蓝图实现。
- 插件对外暴露蓝图节点，对内负责管理本地 TTS 服务。
- 本地 TTS 模型不直接嵌入 UE 进程，而是由插件拉起本地服务进程并通信。
- 第一版只要求支持中文文本输入。
- 第一版只要求生成并播放 wav 音频。

## 5. 第一版验收标准

- 在 UE5.5 C++ 工程中启用插件。
- 在场景或测试蓝图中调用一个蓝图节点，例如 `SpeakText`。
- 输入一段中文，例如“你好，这是测试语音”。
- 插件成功调用本地模型生成语音。
- UE 内部成功播放该语音。
- 整个过程不需要手动启动独立的 TTS 程序。

## 6. 当前固定选型

- 引擎版本：UE5.5
- 开发语言：C++
- 插件形态：项目插件
- 运行平台：Windows
- 推理硬件：本地 NVIDIA 显卡
- TTS 主方案：CosyVoice
- 服务方式：插件内部拉起本地 TTS 服务进程

## 7. 需求变更规则

- 在第一版功能完成前，不新增口型驱动相关需求。
- 在第一版功能完成前，不新增流式生成相关需求。
- 在第一版功能完成前，不切换 UE 版本。
- 在第一版功能完成前，不切换主 TTS 技术路线，除非 CosyVoice 无法在本机稳定运行。
- 后续所有新需求，必须先补充到 `Doc` 文档，再进入开发。
