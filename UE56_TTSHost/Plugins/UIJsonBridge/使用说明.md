# UIJsonBridge 使用说明

`UIJsonBridge` 是一个 UE Editor 插件，用来把 UMG Widget Blueprint 导出成 Codex 可读的 JSON，并把可导入的布局 JSON 再导回 Widget Blueprint。

它的职责是打通 UE 和 AI 协作链路：

- UE 负责承载真实 Widget Blueprint 资产。
- JSON 负责暴露 WidgetTree、布局、样式、事件分析信息。
- Codex 负责阅读、修改和生成 layout JSON。
- 插件负责安全导入 WidgetTree，不自动重建 Blueprint Graph。

## 插件位置

```text
UE56_TTSHost/Plugins/UIJsonBridge
```

工程启用位置：

```text
UE56_TTSHost/UE56_TTSHost.uproject
```

手动编译：

```bat
Build_UE56_TTSHost.bat
```

如果编译失败并提示 `UnrealEditor-UIJsonBridgeEditor.dll` 被占用，先关闭 UE 编辑器再重新编译。

## 随插件分发的 Codex AI Skill

本 UE 插件目录内同时携带一个 Codex 插件包，用于让同事安装后直接通过 AI 制作 UE UI。

Codex 插件目录：

```text
UIJsonBridge/plugins/ui-json-bridge-ai
```

随身 marketplace：

```text
UIJsonBridge/.agents/plugins/marketplace.json
```

内置 Skill：

```text
ue-umg-json-ui-builder
```

安装后可以用类似下面的话触发：

```text
制作 UE 的 UI，根据参考图修改我的 UMG JSON，保持蓝图变量名不变。
```

如果只单独发布 `UIJsonBridge` 这个 UE 插件文件夹，请同时保留：

```text
plugins/ui-json-bridge-ai
.agents/plugins/marketplace.json
```

这样接收方安装 UE 插件后，也能从同一个插件包里安装 Codex AI 插件。

## 导出分级

右键 Widget Blueprint 后，插件提供三种导出入口。

### Export UI JSON - Layout Only

推荐给 Codex 改 UI 布局时使用。

默认文件名：

```text
WBP_Name_layout.json
```

包含：

- WidgetTree 层级
- 控件名称和类型
- Slot 信息
- Canvas Slot 布局
- 可编辑属性和样式
- 资源引用

不包含：

- bindings
- animations
- Blueprint Graph 节点、Pin、连线

`Layout Only` 是最稳定、最推荐回导入 UE 的格式。

### Export UI JSON - Interaction

推荐给 Codex 分析事件响应时使用。

默认文件名：

```text
WBP_Name_interaction.json
```

包含：

- 控件绑定信息
- 动画摘要
- Blueprint Graph 节点摘要
- Event 名称
- Function Call 名称

限制：

- 仅用于分析
- 标记为 `importable=false`
- 导入器会拒绝导入这种 JSON

### Export UI JSON - Full

用于深度分析或备份。

默认文件名：

```text
WBP_Name_full.json
```

包含 layout、bindings、animations、graph nodes、pins、links、comments、defaults 等更完整信息。

注意：`Full` 可以导入其中的 WidgetTree，但目前仍不会重建 Blueprint Graph。日常改布局优先使用 `Layout Only`。

## 推荐协作流程

1. 导出 `_layout.json`。
2. 如果需要理解事件响应，再导出 `_interaction.json`。
3. 把 layout JSON、interaction JSON、参考图或 HTML 一起交给 Codex。
4. Codex 只修改可导入的 layout JSON。
5. 在 UE 中导入修改后的 layout JSON。

推荐提示：

```text
请基于 layout JSON 调整 UI；interaction JSON 只用于理解按钮点击和蓝图调用关系，不要把 interaction JSON 当作可导入文件。
```

## 导入行为

导入会替换目标 Widget Blueprint 的整个 WidgetTree。

当前会导入：

- Widget 层级
- Widget 类和名称
- Display Label
- 可编辑属性
- Panel Slot 属性
- Canvas Slot 布局

当前不会导入：

- Blueprint Graph 节点
- Event 节点
- Pin 连线
- Function Graph
- Animation Timeline

Graph 目前只导出、不导入。自动重建蓝图逻辑风险较高，容易破坏已有事件图。

## 导入安全策略

导入器已经做了几层防护：

- 导入前预检 JSON。
- 拒绝 `importable=false` 文件。
- 拒绝未知 Widget Class。
- 拒绝重复 Widget Name。
- 先在临时 WidgetTree 中构建新树。
- 构建成功后再替换目标 WidgetTree。
- 替换前把旧 WidgetTree 对象移到 transient package，避免 same-name/different-class 崩溃。
- 导入后同步 `WidgetVariableNameToGuidMap`，避免 UMG 编译器 GUID ensure。
- 导入同名同类控件时，会继承原 WidgetTree 中已有变量控件的 `bIsVariable` 标记，避免 AI JSON 漏写变量标记导致蓝图变量丢失。
- 导入后自动编译 Widget Blueprint。

自动化测试覆盖：

- 坏 JSON 导入失败时不破坏原 WidgetTree。
- 真实 responsive safe JSON 可导入。
- LayoutOnly 导出后修改布局，再导入全新 Widget Blueprint。
- 从零生成的 layout JSON 可导入全新 Widget Blueprint。
- 中文 TextBlock 导入后文本正常，并带有效字体。

## 中文与字体

插件导出 JSON 使用 UTF-8 with BOM。导入器读取 JSON 时会强制按 UTF-8 解码，并兼容有 BOM / 无 BOM。

如果详情面板中文字正常、UMG 画布中文字乱码，通常是字体覆盖问题，不是 JSON 编码问题。

导入器会为常见 CJK 文本控件设置可用字体。项目中推荐使用：

```text
/Game/UIJsonBridge/Fonts/UIJB_CJK_Mixed
/Game/UIJsonBridge/Fonts/UIJB_CJK_Mixed_Bold
```

避免回退到：

```text
DroidSansFallback
FontObject=None
```

## 常见问题

### 导入时 UE 崩溃：Cannot replace existing object of a different class

原因：旧 WidgetTree 中存在同名但不同类型的 UObject，UE rename 时可能 fatal。

当前处理：导入器会先把旧 WidgetTree 控件移出，再构建新树，已通过自动化测试覆盖。

### 导入后事件不工作

原因：当前导入只替换 WidgetTree，不重建 Blueprint Graph。

建议：如果要保留事件图，复制原 Widget Blueprint 后导入到副本，并尽量保持控件名称稳定。

### 导入后蓝图提示控件变量不存在

如果 JSON 中能搜到同名控件，但蓝图编译提示变量不存在，通常是该控件没有被标记为 Widget Blueprint 变量。

检查 JSON 中对应控件是否同时满足：

```json
"isVariable": true,
"properties": {
  "bIsVariable": "True"
}
```

控件存在但 `isVariable=false` 或 `bIsVariable=False` 时，UE 不会把它暴露为蓝图变量。

导入器会尽量对同名同类控件继承旧 WidgetTree 的变量标记，但推荐从源头保证 layout JSON 的变量标记正确。

### 不小心导入 interaction JSON

导入器会拒绝，并提示应使用 `Layout Only` 或 `Full` JSON。

### 要给 Codex 改 UI 应该导出什么

只改 UI 时请优先导出：

```text
Export UI JSON - Layout Only
```

需要理解事件时再额外导出：

```text
Export UI JSON - Interaction
```
